# 第 4 项设计：标签（Tags）

## 1. 背景与目标

标签用于建立“跨文档主题索引”，让用户可按主题聚合内容，并为关系图谱提供维度过滤。

目标：

- 支持文档被多个标签归类
- 标签变化可实时同步到协同端
- 标签可参与列表筛选、链接候选排序与后续图谱过滤

## 2. 前置阻塞（必须直说）

当前仍缺“文档实体层”（`docsIndex/currentDocId/doc switch`）。

影响：

- 标签功能可先设计，但实现时必须有“文档级元信息存储位”
- 否则只能给单文档打标签，无法形成全局标签索引

## 3. 范围

### In Scope（本期）

- 文档级标签（`doc.tags: string[]`）
- 标签增删改、去重、归一化
- 标签面板（当前文档标签 + 全局热门标签）
- 按标签过滤文档列表（为后续多文档视图预留）

### Out of Scope（本期不做）

- 段落级标签
- 自动标签抽取（NLP）
- 标签权限/审核流

## 4. 数据结构

逻辑模型（非实现代码）：

### 4.1 文档标签字段

- 在文档元信息中增加：`tags: string[]`
- 单个标签建议：
  - 原始值：`rawTag`
  - 归一化值：`normalizedTag`

### 4.2 全局标签索引（共享）

- 位置：`Y.Doc` 根下 `tagIndex`（`Y.Map`）
- Key：`normalizedTag`
- Value：
  - `docIds: string[]`
  - `count: number`
  - `updatedAt: number`

说明：`count` 可由 `docIds` 计算，也可冗余存储提高读取速度。

### 4.3 与内链/别名关系

- 标签不参与 `docId` 唯一解析，不替代标题/别名
- 可作为候选排序因子（后续迭代）

## 5. 状态流与交互流

1. 用户在当前文档添加标签
2. 标签归一化与去重
3. 写回当前文档 `tags`
4. 增量更新 `tagIndex`
5. 其他客户端实时收到变更
6. 标签面板与筛选结果刷新

删除标签流程反向执行。

## 6. 边界场景与异常处理

1. 大小写/空白差异：统一归一化（如 `AI` 与 ` ai `）
2. 同一文档重复输入标签：只保留一条
3. 标签过长：拒绝（建议上限 32）
4. 特殊字符标签：按规则校验（见“待你拍板项”）
5. 删除最后一个引用文档后：`tagIndex` 对应项自动移除

## 7. 验收标准（可测试）

1. 为文档 A 添加标签 `产品` 后，标签面板显示 `产品`，且 `count >= 1`
2. 文档 B 添加同标签后，全局计数正确增加
3. 删除 A 的该标签后，全局计数正确减少
4. 多端协同下，一端改标签，另一端实时同步
5. 同文档重复添加同标签不会产生重复记录

## 8. 与后续 MySQL 对齐点

建议表：

- `document_tags(id, doc_id, tag_text, normalized_tag, updated_at)`

索引建议：

- `idx_tag_normalized(normalized_tag)`
- `idx_tag_doc(doc_id)`
