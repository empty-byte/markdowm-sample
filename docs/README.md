# 设计文档目录说明

本目录用于沉淀每个需求项的设计，不直接放实现代码。

## 命名规范

- 文件名格式：`YYYY-MM-DD-<序号>-<topic>-design.md`
- 示例：`2026-03-30-01-internal-link-design.md`

## 每个设计文档必填章节

1. 背景与目标
2. 范围（In Scope / Out of Scope）
3. 数据结构
4. 状态流与交互流
5. 边界场景与异常处理
6. 验收标准（可测试）
7. 与后续 MySQL 的对齐点

## 当前实现优先级（已调整）

1. 内嵌网页
2. 白板
3. 内链（`[[文档名]]`）
4. 双向链接
5. 别名
6. 标签
7. 关系图谱

说明：现有设计文档文件名中的序号保留历史记录含义，不代表最新实施顺序。

## 记录规则

- 每新增一项需求，先补对应设计文档，再进入编码阶段。
- 若需求变更，更新原设计文档并在文末追加“变更记录”。
