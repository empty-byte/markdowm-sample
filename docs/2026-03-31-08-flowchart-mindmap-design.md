# 第 8 项设计：流程图 + 思维导图（方案 2，类飞书）

## 1. 背景与目标

在当前编辑器（Vue3 + Milkdown + Yjs + Hocuspocus）基础上，新增“流程图”和“思维导图”两类结构化图形能力，交互目标对齐飞书文档：

- 通过 `/` 菜单插入图形
- 在正文内以卡片方式展示预览
- 提供悬浮操作（编辑、删除）
- 编辑保存后，正文卡片即时刷新

已确认决策：

- 采用 **方案 2（双引擎可视化）**
  - 流程图：`LogicFlow`
  - 思维导图：`simple-mind-map`

## 2. 范围

### In Scope（本期）

- `/` 菜单新增“流程图”“思维导图”入口
- 各自弹框编辑器（与白板弹框交互一致）
- Markdown Token 持久化引用
- 正文内卡片渲染（预览 + 编辑/删除）
- 刷新后可恢复（先本地存储，后续可迁移）

### Out of Scope（本期不做）

- 流程图/思维导图实时多人同图协作
- 图权限系统（只读、共享范围、审批）
- 图版本分支与冲突合并

## 3. 技术选型与理由

## 3.1 流程图：LogicFlow

- Vue 集成成本低
- 节点/边模型清晰，适合流程图语义
- 后续可扩展泳道、条件分支等业务组件

## 3.2 思维导图：simple-mind-map

- 脑图交互成熟（折叠、展开、层级编辑）
- 数据结构天然树形，适配思维导图场景
- 易做导出预览图

## 3.3 不采用 Mermaid 作为主方案的原因

- Mermaid 更偏“文本画图”，与飞书式拖拽编辑体验存在差异
- 在高频编辑场景下，学习成本与可视化反馈不如双引擎

## 4. Markdown 语法约定（建议）

建议延续白板模式，采用图片语法 + 自定义协议：

- 流程图：`![flowchart:标题](flowchart://<diagramId>)`
- 思维导图：`![mindmap:标题](mindmap://<diagramId>)`

说明：

- 文档正文只保留轻量引用（`diagramId`）
- 图的完整 JSON 数据放外部存储（本期先 localStorage，后续 Y.Doc/MySQL）

## 5. 数据结构

## 5.1 统一图数据模型（逻辑层）

- `id: string`（建议 ULID）
- `type: 'flowchart' | 'mindmap'`
- `title: string`
- `scene: object`（对应各自引擎 JSON）
- `previewUrl: string`（PNG dataURL 或静态地址）
- `updatedAt: number`

## 5.2 前端存储（本期）

- `localStorage` 分桶（建议）：
  - `milkdown:flowcharts:v1`
  - `milkdown:mindmaps:v1`

说明：与现有白板策略保持一致，先跑通端内体验。

## 5.3 后续 MySQL 对齐（预留）

建议统一落库表（避免三套白板/流程图/脑图各自分裂）：

- `doc_diagrams(id, doc_id, type, title, scene_json, preview_url, updated_at, created_by)`

## 6. 状态流与交互流

## 6.1 插入流程

1. 用户输入 `/`，选择“流程图”或“思维导图”
2. 打开对应编辑弹框
3. 用户编辑并点击保存
4. 生成 `diagramId`，保存 `scene + preview`
5. 在当前光标插入对应 Markdown token
6. NodeView 识别 token，渲染图卡片

## 6.2 编辑流程

1. hover 卡片出现“编辑/删除”
2. 点击编辑，按 `diagramId` 读取已存 scene
3. 打开对应编辑弹框并回填
4. 保存后更新 `scene + preview + updatedAt`
5. 强制触发渲染刷新（key 包含 `updatedAt`）

## 6.3 删除流程

1. 点击删除，仅删除正文 token
2. 图数据先不立即硬删除（避免误删）
3. 后续可增加 orphan 清理策略（定时或手动）

## 7. 编辑器内渲染设计

建议新增两套 NodeView 插件：

- `flowchart-node-view.ts`
- `mindmap-node-view.ts`

行为与白板保持一致：

- token 渲染为卡片
- 原 token 文本隐藏
- 卡片顶部展示标题与 ID/来源说明
- hover 显示操作按钮

## 8. 边界场景与异常处理

1. token 存在但数据不存在：显示“数据缺失”占位，提供“编辑重建”
2. 导出预览失败：保留 scene，提示“预览生成失败，可继续保存”
3. 大图性能问题：限制导出分辨率与图尺寸
4. 渲染不刷新：NodeView key 必须包含 `updatedAt`
5. 非法 token：按普通文本处理，不中断编辑
6. 协同下同时编辑同一图（本期无协作）：以最后保存覆盖（LWW）

## 9. 验收标准（可测试）

1. `/` 菜单可见“流程图”“思维导图”入口
2. 插入后，Markdown 中出现对应协议 token（`flowchart://` / `mindmap://`）
3. 正文卡片可见预览图，且紧随插入位置显示
4. 点击编辑后保存，正文预览即时刷新
5. 点击删除后，正文中卡片消失且 token 被移除
6. 刷新页面后，可通过 `diagramId` 恢复并渲染

## 10. 与现有白板能力的关系

- 白板继续保留（自由画布）
- 流程图/思维导图是结构化图形补充
- 三者统一使用“token + 外部数据 + 卡片 NodeView”架构，降低维护成本

## 11. 实施阶段建议（先不编码）

3. 第三阶段：统一抽象 `diagram` 存储层，减少重复逻辑
4. 第四阶段：评估是否把图数据迁入 Y.Doc / MySQL

## 16. 流程图节点库增强（方案 1 + 方案 2 部分）

### 16.1 目标

- 在流程图编辑器中提供“更多节点选择”（对齐飞书的基础体验）
- 新增“分类分组”展示
- 支持“点击插入”和“拖拽到画布插入”

### 16.2 本次实现范围

- 左侧节点库分组：
  - 流程图
  - 基础图形
  - BPMN
  - 辅助
- 节点总数扩展为 20+（包含基础流程节点、基础几何节点、BPMN 常用节点、辅助节点）
- 拖拽交互：
  - 从节点库拖到画布任意位置后释放即可插入
  - 拖拽中画布显示投放态高亮提示
- 保留原有点击插入

### 16.3 数据结构与状态

- `paletteGroups`: 节点分组定义（key + label）
- `paletteItems`: 节点模板定义（key / label / hint / group / template）
- `paletteItemMap`: key 到模板的索引映射（用于点击/拖拽快速查找）
- `canvasDropActive`: 当前是否处于拖拽投放态
- `draggingPaletteKey`: 当前被拖拽的节点 key

### 16.4 状态流

1. 用户在节点库点击节点 -> 根据 `paletteItemMap` 取模板 -> 自动网格位置插入
2. 用户拖拽节点 -> 写入 `dataTransfer` -> 拖到画布时进入投放态
3. 在画布释放 -> 根据鼠标坐标换算画布坐标 -> 在释放点插入对应节点
4. 插入后更新节点计数种子，继续支持后续自动排布

### 16.5 边界场景

- 拖拽数据丢失或 key 无效：忽略本次 drop，不抛异常
- 坐标换算失败：回退到自动排布插入
- 保存中（`saving=true`）：禁止新增节点与拖拽插入
- BPMN 节点未注册：会导致类型不可识别，因此初始化阶段强制注册扩展

### 16.6 验收标准

1. 节点库按 4 类分组展示，节点数量明显大于原先版本
2. 点击任一节点可插入到画布
3. 拖拽任一节点到画布可在释放点插入
4. 放大/缩小仍作用于整个画布（所有节点与连线同步缩放）
5. 不影响既有“导出图片 / 查看 JSON / 撤销重做 / 自适应”等工具能力

## 18. 节点属性编辑（2026-04-01）

### 18.1 目标

- 支持对单个节点进行样式和尺寸编辑
- 核心覆盖：文字颜色、文字大小、背景色、边框色、边框粗细、宽高、圆角

### 18.2 交互设计

- 在流程图编辑弹框右侧新增“节点属性”面板
- 点击画布中的任意节点后，右侧面板回填当前节点属性
- 修改属性后实时应用到当前选中节点
- 点击画布空白处取消选中，面板回到提示态

### 18.3 数据与状态

- `selectedNodeId`: 当前选中节点 ID
- `selectedNodeType`: 当前选中节点类型
- `nodeStyleForm`: 属性表单（text/textColor/fontSize/fill/stroke/strokeWidth/width/height/radius）

### 18.4 状态流

1. `node:click` -> 读取节点模型与文本 -> 回填 `nodeStyleForm`
2. 面板字段变更 -> 归一化数值 -> `setProperties` + `updateText`
3. `blank:click` / 选中节点被删除 -> 清空选中态
4. `node:resize` 时若该节点处于选中态 -> 同步刷新表单尺寸

### 18.5 边界场景

- 非法输入（空值/非数值）自动回退到安全默认值
- 文本节点禁用宽高编辑
- 非矩形类节点禁用圆角编辑
- 保存中（`saving=true`）禁用属性交互

### 18.6 验收标准

1. 选中节点后可见并可编辑文字颜色/字号/背景/边框/边框粗细
2. 选中节点后可编辑宽度和高度（文本节点除外）
3. 矩形类节点可编辑圆角
4. 样式改动立即反映在画布节点上
5. 点击空白后属性面板回到未选中提示态
