# 第 5 项设计：关系图谱（Graph）

## 1. 背景与目标

关系图谱用于把文档网络可视化，帮助用户快速理解“当前文档与周边知识节点”的连接结构。

本项目标（MVP）：

- 先做文档级节点（不做段落级）
- 先做当前文档 1-hop 子图（当前文档 + 直接入链/出链）
- 支持点击节点跳转文档
- 支持按状态过滤边（resolved / ambiguous / dangling）

## 2. 前置阻塞（必须直说）

当前存在两类前置：

1. 代码层前置：缺少“文档实体层”（`docsIndex/currentDocId/doc switch`）
2. 数据层前置：`linkEdges` 仍是设计态，未落地实现

影响：

- 若直接做图谱 UI，会陷入“只有假数据可画图”
- 推荐先完成最小文档实体层 + 可用链接边数据，再做图谱渲染

## 3. 范围

### In Scope（本期）

- 基于 `docId` 的文档级节点
- 基于 `linkEdges` 的有向边
- 1-hop 视图：
  - 节点集合 = 当前文档 + 入邻居 + 出邻居
  - 边集合 = 与当前文档相关的直接边
- 基础交互：缩放、拖拽、点击跳转、hover 提示

### Out of Scope（本期不做）

- 全量全图布局（大图）
- 自动聚类 / 社区发现
- 段落锚点级图谱
- 时间轴图谱回放

## 4. 数据结构

逻辑模型（非实现代码）：

### 4.1 节点

- `id: docId`
- `label: title`（可回退为 docId 短码）
- `meta: { aliasesCount, tagsCount, updatedAt }`

### 4.2 边

- `id: edgeId`
- `source: sourceDocId`
- `target: targetDocId | null`
- `state: resolved | ambiguous | dangling`
- `rawLinkText: string`

说明：`target = null` 的边可渲染为“悬空边”（指向虚拟节点或特殊样式）。

### 4.3 图视图模型（运行时）

- `GraphViewModel = { nodes: NodeVM[], edges: EdgeVM[], focusDocId }`
- 由 `docsIndex + linkEdges` 计算生成

## 5. 状态流与交互流

1. 进入某文档（`currentDocId`）
2. 从 `linkEdges` 计算当前文档 1-hop 邻域
3. 映射为 `GraphViewModel`
4. 图谱面板渲染
5. 用户点击节点 -> 切换文档上下文 -> 图谱重算

过滤流：

1. 用户切换“仅看 resolved / 显示 dangling”等过滤器
2. 在 VM 层过滤边与关联节点
3. 局部重绘

## 6. 边界场景与异常处理

1. 孤立文档：只显示单节点并提示“暂无连接”
2. 自链接：渲染为自环边
3. 悬空边过多：限制展示数量并提供“查看全部”入口
4. 节点标题缺失：回退显示 docId 短码
5. 超过阈值（如节点 > 200）：自动退化为列表模式或提示缩小范围

## 7. 验收标准（可测试）

1. 当前文档存在出链时，图中可见目标节点与有向边
2. 当前文档存在入链时，图中可见来源节点与有向边
3. 切换过滤器后，边和节点可按预期变化
4. 点击节点可完成文档切换，并触发图谱刷新
5. 孤立文档场景下图谱仍可正常显示（不报错）

## 8. 与后续 MySQL 对齐点

图谱读取接口可直接来源于：

- `documents`
- `document_links`

后续可增加预聚合表（可选）：
